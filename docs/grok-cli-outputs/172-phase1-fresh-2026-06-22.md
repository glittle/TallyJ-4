# Issue 172 Phase 1 — Election State Management

**Branch:** `cli5`  
**Date:** 2026-06-22  
**Scope:** Minimal backend stage guards + TypeScript type sync (no UI, SignalR changes, or assistant-teller restrictions)

## Summary

Added a fourth election stage (`Finalized`) with guarded transitions in `ChangeElectionStageAsync`. Stage changes are restricted to **Full Tellers at the API layer** (`FullTellerAccess` policy); business-rule guards inside the service apply uniformly to all authorized callers.

## Backend Changes

| Area | Change |
|------|--------|
| `ElectionStage` enum | Added `Finalized` after `ProcessingBallots` |
| `FullTellerAccess` policy | New authorization handler — global `Admin`, election `Owner`/`Admin` join roles; guest tellers denied |
| `ElectionsController` | `PUT …/stage` now uses `[Authorize(Policy = "FullTellerAccess")]` |
| `ElectionStageTransitions` | Sequential forward moves, free backward moves, no skipping |
| `ElectionStageFinalizationReadiness` | Analysis complete, no outstanding ballot issues, ties resolved; ballot status check runs only when a final summary exists and uses client-side evaluation to stay SQLite-compatible |
| `ChangeElectionStageDto` | `ConfirmLeavingFinalized` flag for reverting from `Finalized` |
| `ChangeElectionStageResult` | Result type with `IsSuccess`, `RequiresConfirmation`, `ErrorMessage` |
| `ElectionService` | Removed in-service Full Teller role branching; guards apply directly |

### Guard Rules (all authorized callers)

1. **Transition validity** — cannot skip stages or set the same stage.
2. **Entering Finalized** — blocked unless finalization readiness passes.
3. **Leaving Finalized** — returns `RequiresConfirmation` unless `confirmLeavingFinalized: true`.

Authorization ensures only Full Tellers reach these guards; the service no longer duplicates role checks.

### Logging

- `LogWarning` on rejected transitions and failed finalization
- `LogInformation` on successful stage changes (service + controller) with previous/new stage and confirmation flag
- `FullTellerAccessHandler` logs authorization decisions

## API Contract Notes

- `PUT /api/elections/{guid}/stage` requires Full Teller authorization (403 if denied).
- Body accepts optional `confirmLeavingFinalized: boolean`.
- Leaving `Finalized` without confirmation → **409 Conflict**.
- Blocked finalization or invalid transition → **400 Bad Request**.

## Cleanup (2026-06-22)

- Moved Full Teller enforcement to `FullTellerAccess` authorization policy; removed `IsFullTellerForElectionAsync` from `ElectionService`.
- Removed role-based guard bypass inside `ChangeElectionStageAsync`.
- Fixed `ElectionStageFinalizationReadiness` to skip live ballot queries when no final summary exists (prevented SQLite translation error).
- Consolidated tie-resolution checks into a single expression.
