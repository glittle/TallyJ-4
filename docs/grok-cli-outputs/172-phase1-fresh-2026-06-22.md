# Issue 172 Phase 1 — Election State Management (Fresh Start)

**Branch:** `cli5`  
**Date:** 2026-06-22  
**Scope:** Minimal backend stage guards + TypeScript type sync (no UI, SignalR changes, or assistant-teller restrictions)

## Summary

Added a fourth election stage (`Finalized`) with guarded transitions in `ChangeElectionStageAsync`. Full Tellers (global Admin, election Owner/Admin join records) bypass all guards. Non–Full Tellers must meet finalization readiness checks and confirm when leaving `Finalized`.

## Backend Changes

| Area | Change |
|------|--------|
| `ElectionStage` enum | Added `Finalized` after `ProcessingBallots` |
| `ElectionStageTransitions` | New helper: sequential forward moves, free backward moves, no skipping |
| `ElectionStageFinalizationReadiness` | New evaluator: analysis complete (`Result` + `ResultSummary` type `F` with `UseOnReports`), no outstanding ballot issues, all ties resolved |
| `ChangeElectionStageDto` | Added `ConfirmLeavingFinalized` flag |
| `ChangeElectionStageResult` | New result type with `IsSuccess`, `RequiresConfirmation`, `ErrorMessage` |
| `IElectionService` / `ElectionService` | `ChangeElectionStageAsync` now accepts DTO, applies guards, logs transitions |
| `ElectionsController` | Returns 409 Conflict when confirmation required; 400 for blocked transitions |

### Guard Rules (non–Full Tellers only)

1. **Transition validity** — cannot skip stages or set the same stage.
2. **Entering Finalized** — blocked unless finalization readiness passes.
3. **Leaving Finalized** — returns `RequiresConfirmation` unless `confirmLeavingFinalized: true`.

Full Tellers are never blocked by any of the above.

### Logging

- `LogWarning` on rejected transitions and failed finalization
- `LogInformation` on successful stage changes (service + controller) with previous/new stage, Full Teller status, and confirmation flag

## API Contract Notes

- `PUT /api/elections/{guid}/stage` body now accepts optional `confirmLeavingFinalized: boolean`.
- Leaving `Finalized` without confirmation → **409 Conflict** with message in `ApiResponse.error`.
- Blocked finalization or invalid transition → **400 Bad Request**.
